# Projekt-Regeln

## Arbeitsweise
- Immer autonom arbeiten, keine Rückfragen stellen
- Selbstständig mitdenken und Verbesserungen umsetzen
- Probleme eigenständig analysieren und lösen
- Das Projekt bei jeder Session so gut wie möglich weiterbringen

## Code-Qualität
- Immer neueste Library-Versionen und Best Practices (2025) verwenden
- Robustes Error-Handling in jeder Funktion
- Logging einbauen (wichtige Events, Fehler, Warnungen)
- Clean Code: lesbar, gut strukturiert, dokumentiert
- Performance optimieren wo möglich
- Tests schreiben wo sinnvoll

## Sicherheit
- API Keys und Secrets ausschließlich in .env Dateien
- .gitignore muss alle sensitiven Dateien ausschließen
- Keine hardgecodeten Credentials
- Input Validation überall

## Proaktive Verbesserungen
- Wenn etwas fehlt oder verbessert werden kann: einfach machen
- Veraltete Dependencies aktualisieren
- Code-Duplikation entfernen
- Fehlende Dokumentation ergänzen
- Sinnvolle Features vorschlagen und umsetzen

## Lieferung
- Immer Pull Request erstellen
- PR-Beschreibung auf Deutsch
- Zusammenfassen: Was wurde gemacht, was wurde verbessert, was kommt als nächstes
